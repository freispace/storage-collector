# freispace Storage Collector

Cross-platform desktop tray app (Tauri 2 + Svelte 5 + Rust). Walks local folders, sums file sizes, and POSTs daily storage statistics to the freispace REST API. Runs silently in the system tray.

## Package manager

Always use **pnpm**, never npm or yarn.

```bash
pnpm install
pnpm dev           # full stack: Vite (port 1420) + Rust backend
pnpm dev:frontend  # Vite only, no Rust
pnpm build         # native installer → src-tauri/target/release/bundle/
pnpm check         # svelte-check (TypeScript in .svelte files)
```

## Stack

- **Frontend:** Svelte 5 (runes), TypeScript, Tailwind CSS v4, DaisyUI v5, Vite 6
- **Backend:** Rust, Tauri 2, SQLite (sqlx), tokio-cron-scheduler, reqwest (rustls)
- **Bindings:** tauri-specta auto-generates `src/bindings.ts` on every debug run

## Architecture

```
src/                     # Svelte/TypeScript frontend
  lib/api.ts             # All invoke() wrappers + event listeners (hand-maintained)
  lib/stores/            # Svelte 5 rune-based state (settings, logs)
  components/            # UI: TabBar, SettingsTab, LogsTab and sub-components
src-tauri/src/
  lib.rs                 # AppState, Tauri builder, command + event registration
  commands/              # Tauri commands: settings, folders, projects, scheduler, logs
  scheduler/             # Cron jobs: run_tick (collect+submit), retry_pending
  api/                   # FreispaceClient (reqwest), SubmissionRateLimiter, DTOs
  db/                    # sqlx: models, queries, migrations
  tray/                  # TrayManager (5 icon states), context menu
  error.rs               # AppError (thiserror + Serialize for Tauri propagation)
  events.rs              # LogEntryEvent, SchedulerTickEvent
```

## Key design decisions

**TypeScript bindings** — `src/bindings.ts` is auto-generated by tauri-specta; do not edit it. `src/lib/api.ts` is the hand-maintained wrapper with all type definitions (currently the bindings file is a stub placeholder).

**Scheduler rebuilds** — Changing api_key, global_schedule, auto_run, or any folder config tears down and recreates the `JobScheduler` via `rebuild_scheduler()`. This is intentional.

**sqlx offline mode** — `.sqlx/` query metadata is committed. CI uses `SQLX_OFFLINE=true`. After adding or changing any SQL query, regenerate:
```bash
cd src-tauri
$env:DATABASE_URL="sqlite://.sqlx-prepare.db?mode=rwc"; cargo sqlx prepare
# commit the updated .sqlx/ directory
```

**Rate limiter** — `SubmissionRateLimiter` caps API submissions at 1 req/sec. List/fetch endpoints bypass it.

**No environment variables** — All config (API key, schedule, folders) is stored in SQLite via the UI. DB path: `{OS app data dir}/com.freispace.storage-collector/storage-collector.db`.

**Window hide, not close** — `CloseRequested` hides the window to tray. Only the tray "Quit" menu item calls `app.exit(0)`.

## Rust workflow

```bash
cd src-tauri
cargo clippy -- -D warnings   # lints (CI treats warnings as errors)
SQLX_OFFLINE=true cargo test  # tests (use offline mode; no live DB needed)
```

Rust MSRV is **1.77.2**. Release profile uses `opt-level = "z"`, `lto = true`, `strip = true`.

## Adding Tauri commands

1. Add the `#[tauri::command]` + `#[specta::specta]` function in the appropriate `commands/*.rs` file
2. Register it in `lib.rs` (both the `invoke_handler` and the specta builder)
3. Run `pnpm dev` once — tauri-specta regenerates `src/bindings.ts`
4. Add the typed wrapper to `src/lib/api.ts`

## CI

Matrix: Windows, macOS, Ubuntu. Runs `cargo test`, `cargo clippy -- -D warnings`, and `vite build`. All use `SQLX_OFFLINE=true`.

Release builds are triggered by `v*.*.*` tags and produce native installers via `tauri-apps/tauri-action`. The updater `pubkey` in `tauri.conf.json` must be set to the actual signing key before releases work end-to-end.
